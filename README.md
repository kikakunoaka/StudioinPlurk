# 噗浪周邊客製工作室｜評價目錄

純前端（HTML／CSS／JS，無需後端、無需 API Key）的廠商評價網站，
資料即時讀取自 Google 試算表：

https://docs.google.com/spreadsheets/d/1fpoLejfAJz6bQwAPA4xwTzQzW21Y-5ABPT1f-vaGwsQ/

## 網站結構

- `index.html` — 主頁
  - 最上方為可折疊的「更新日誌」，內容讀取「更新日誌」分頁（每一列都是一筆
    紀錄，第一欄為日期、其餘欄為說明，無標題列）
  - 篩選搜尋器讀取「sources」分頁：第 2 列（A2:E2）為搜尋項目標題、
    第 3 列起（A3:E）為每個項目底下的可篩選內容；項目標題須與
    「工作室列表」中對應的欄位標題完全相同，篩選才會生效
  - 列表讀取「工作室列表」分頁的全部欄位內容（名稱欄除外的欄位皆會顯示
    成卡片上的 TAG），資料從第 2 列（第一位工作室）開始
  - 每張卡片：ICON 欄位的圖片會顯示在卡片左側；「體驗評價」欄位的內容
    會固定顯示在卡片右上角的圓形勳章、不會顯示為一般 TAG
  - 任何欄位內容若包含半形逗號 `,`，會自動拆解成多個 TAG
    （例如 `常態團,獨立單` → `常態團`、`獨立單`）
- `studio.html` — 各工作室分頁模板，透過網址 `?name=工作室名稱`
  讀取「與工作室同名」的分頁：
  - 廠商介紹：讀取 A1:G16，A 欄為欄位標題、B～G 欄為內容
    - 第 1 列（A1 為標題、B1 起為內容）視為 ICON 圖示列，內容需為圖片網址，
      會顯示成小圖示而非文字 TAG
    - 第 2～16 列，每個標題底下的內容以橢圓 TAG 呈現；內容若包含半形逗號
      會自動拆解成多個 TAG
  - 體驗者感想：從第 18 列開始（第 18 列為標題列，19 列起為資料）
- `js/config.js` — 設定檔（試算表 ID、分頁名稱、篩選欄位、讀取範圍）
- `js/sheets.js` — 讀取 Google 試算表的共用函式
- `js/main.js` / `js/studio.js` — 各頁面邏輯
- `css/style.css` — 樣式

## 第一步：確認 Google 試算表分享設定

共用 →「知道連結的任何人」→ 權限「檢視者」。

## 第二步：上傳到 GitHub 並開啟 GitHub Pages

1. 在 GitHub 建立 repository，把資料夾內所有檔案上傳（保留 `css/`、`js/` 資料夾結構）。
2. Settings → Pages → Source 選 `Deploy from a branch`，Branch 選你的預設分支、
   資料夾選 `/ (root)`，Save。
3. 等 1～2 分鐘取得網址，例如 `https://你的帳號.github.io/repo名稱/`。

## 第三步：日後新增工作室怎麼做？

1. 新增一個分頁，**分頁名稱需與工作室名稱完全一致**。
2. 在該分頁填寫：
   - A1 填寫「ICON」之類的標題，B1（可延伸至 C1:G1）填入圖示圖片網址，
     會顯示成小圖示
   - A2:A16 填寫其餘欄位標題（例如：平台、委託方式、工廠、客製項目…）
   - 對應列的 B～G 欄填入內容（同一標題可填多個值：可分別填在不同欄，
     或填在同一格用半形逗號 `,` 分隔，例如 `常態團,獨立單`，都會各自變成
     一個 TAG）
   - 第 18 列填寫體驗回報的標題（例如：日期、暱稱、心得、評分），
     第 19 列起逐筆填入體驗者感想
3. 在「工作室列表」分頁新增一列：填入名稱、ICON 圖片網址、體驗評價，
   以及其他分類欄位（同樣支援半形逗號分隔多個值），會自動出現在主頁卡片。
4. 若該分類欄位也要出現在主頁的篩選搜尋器，請到「sources」分頁：
   在對應欄的第 2 列填入與「工作室列表」完全相同的欄位標題，
   第 3 列起填入可供篩選的選項內容。
5. 不需要改任何程式碼、不需要重新部署，重新整理網站即可看到新工作室。

## 更新日誌分頁格式

「更新日誌」分頁建議格式：第一欄為日期，其餘欄位為更新內容說明，
每一列（含第一列）都會顯示為一筆日誌紀錄，**不需要標題列**。

## sources 分頁格式（主頁篩選搜尋器）

- 第 2 列（A2:E2）：搜尋項目標題，需與「工作室列表」中對應欄位的
  標題文字完全相同（例如「營運狀況」、「委託方式」）。
- 第 3 列起（A3:E）：該項目底下可供勾選的選項內容，每一格一個選項，
  同一格內也可用半形逗號分隔多個選項。

## 欄位對應與客製

若試算表分頁名稱、欄位標題文字或讀取範圍與此文件假設的不同，
請至 `js/config.js` 修改對應設定值（`listSheetName`、`changelogSheetName`、
`sourcesSheetName`、`iconColumnName`、`ratingColumnName`、`profileRange`、
`reviewStartRow`）。

## 本機預覽

`fetch()` 讀取外部資料在 `file://` 協定下可能被瀏覽器 CORS 政策擋下，
本機測試建議：

```bash
python3 -m http.server 8000
```

再開啟 `http://localhost:8000`。上傳到 GitHub Pages 後就沒有這個限制。

## 已知限制

- Google 試算表的公開查詢端點偶爾會有幾秒快取延遲，屬正常現象。
- 若某工作室分頁尚未填寫 A1:G16 或第 18 列以後的內容，對應區塊會顯示
  「尚未填寫」或「目前還沒有體驗心得」等提示文字，不會顯示錯誤。
