# 工作室評價目錄

一個純前端（HTML／CSS／JS，無需後端、無需 API Key）的廠商評價網站，
資料即時讀取自你的 Google 試算表：

https://docs.google.com/spreadsheets/d/1fpoLejfAJz6bQwAPA4xwTzQzW21Y-5ABPT1f-vaGwsQ/

## 網站結構

- `index.html` — 主頁，讀取「工作室列表」分頁，卡片 + 上方 TAG 篩選列
- `studio.html` — 各工作室分頁模板，透過網址參數 `?name=工作室名稱` 決定要讀哪個分頁
  - 讀取「與工作室同名」的分頁 → 廠商介紹區域
  - 讀取「體驗回報」分頁，篩選出屬於該工作室的列 → 體驗者感想區域
- `js/config.js` — 設定檔（試算表 ID、分頁名稱）
- `js/sheets.js` — 讀取 Google 試算表的共用函式
- `js/main.js` / `js/studio.js` — 各頁面邏輯
- `css/style.css` — 樣式

## 第一步：確認 Google 試算表分享設定

因為網站是純前端直接向 Google 讀取資料，試算表**必須**設定為：

> 共用 →「知道連結的任何人」→ 權限「檢視者」

（不需要「發布到網路」，一般的連結分享即可。）

## 第二步：上傳到 GitHub 並開啟 GitHub Pages

1. 在 GitHub 建立一個新的 repository（例如 `studio-reviews`）。
2. 把這個資料夾內所有檔案上傳（可直接在 GitHub 網頁介面拖曳上傳，或用 git push）。
3. 到 repository 的 **Settings → Pages**。
4. 「Source」選擇 `Deploy from a branch`，Branch 選 `main`（或你的預設分支），資料夾選 `/ (root)`，按 Save。
5. 等 1～2 分鐘，GitHub 會提供一個網址，例如：
   `https://你的帳號.github.io/studio-reviews/`
6. 打開該網址即可看到主頁；點擊任一工作室卡片會進入
   `studio.html?name=工作室名稱`。

## 第三步：日後新增工作室怎麼做？

1. 在 Google 試算表中，新增一個分頁，**分頁名稱需與工作室名稱完全一致**
   （例如新增「貓貓窩」，分頁就叫「貓貓窩」）。
2. 在「工作室列表」分頁新增一列資料，第一欄填入同樣的名稱「貓貓窩」，
   其餘欄位依照現有欄位（營運、委託方式、平台、工廠、項目、品項、
   特殊工藝、材質…等）填入分類資訊 — 這些會自動變成主頁的 TAG。
3. 不需要改任何程式碼、不需要重新部署，重新整理網站即可看到新工作室。

## 關於「體驗回報」分頁

建議此分頁其中一欄用來記錄「工作室名稱」（例如透過 Google 表單蒐集
心得回饋，表單中設一題「請選擇工作室」）。網站會自動抓出該欄值等於
目前工作室名稱的所有列，顯示在「體驗者感想」區塊。

## 欄位對應與客製

- 主頁的 TAG 分類：預設「工作室列表」除了第一欄（名稱）以外的欄位都會
  變成可篩選的 TAG。若某欄位其實是「連結」「介紹」「備註」等補充說明，
  而不想被當成 TAG，可以到 `js/config.js` 的 `nonTagKeywords` 加入
  該欄位名稱中會出現的關鍵字。
- 若試算表分頁名稱與此文件假設的不同，請至 `js/config.js` 修改
  `listSheetName` / `reviewSheetName`。

## 本機預覽

由於使用了 `fetch()` 讀取外部資料，用瀏覽器直接開啟 `index.html`
（`file://` 協定）在部分瀏覽器會被 CORS 政策擋下。建議本機測試時
用一個簡單的本地伺服器，例如：

```bash
python3 -m http.server 8000
```

再開啟 `http://localhost:8000`。上傳到 GitHub Pages 後就沒有這個限制。

## 已知限制／注意事項

- 我目前無法直接讀取試算表實際儲存格內容（Google 試算表需要登入或
  JavaScript 才能顯示），所以「工作室列表」「小羊派」「體驗回報」這幾個
  分頁的實際欄位名稱是依照你描述與「sources」分頁的分類選項（營運、
  委託方式、平台、工廠、項目、品項、特殊工藝、材質）推測設計的彈性
  程式邏輯 — 無論欄位怎麼排都能運作，但建議你部署後實際打開網站
  檢查一次，若某些欄位顯示位置不理想，告訴我實際欄位名稱，我可以
  幫你微調對應邏輯。
- Google 試算表的 `gviz` 公開端點偶爾會有幾秒的快取延遲，屬正常現象。
